Installation Guide

Last updated: May 1, 2025 By: ZeroTrustKerberosLink Team

ZeroTrustKerberosLink is designed to be easy to install while maintaining enterprise-grade security. This guide will walk you through the installation process step by step.

Prerequisites

  • Linux-based server (Ubuntu 20.04+ or RHEL 8+ recommended)
  • Docker and Docker Compose
  • Access to your Kerberos KDC
  • AWS IAM permissions for integration

Step 1: Download the Installation Package

curl -sSL https://download.zerotrustkerberos.link/latest | sudo bash

Step 2: Configure Your Environment

Create a configuration file by copying the example:

cp /etc/zerotrustkerberos/config.example.yaml /etc/zerotrustkerberos/config.yaml
nano /etc/zerotrustkerberos/config.yaml

Step 3: Set Up Kerberos Integration

Configure your Kerberos settings in the configuration file:

# Kerberos Configuration
kerberos:
  realm: YOUR.REALM
  kdc: kdc.your-domain.com
  admin_server: admin-server.your-domain.com
  keytab_path: /etc/zerotrustkerberos/krb5.keytab

Step 4: Configure AWS Integration

Set up your AWS integration settings:

# AWS Configuration
aws:
  region: us-west-2
  role_mapping:
    - kerberos_principal: "admin@YOUR.REALM"
      aws_role: "arn:aws:iam::123456789012:role/AdminRole"
    - kerberos_principal: "developer@YOUR.REALM"
      aws_role: "arn:aws:iam::123456789012:role/DeveloperRole"

Step 5: Configure Security Settings

Enable security features based on your requirements:

# Security Configuration
security:
  input_validation: true
  xss_protection: true
  content_security_policy: true
  secure_headers: true
  rate_limiting:
    enabled: true
    requests_per_minute: 60

Step 6: Start the Service

sudo systemctl enable zerotrustkerberos
sudo systemctl start zerotrustkerberos

Step 7: Verify Installation

Check that the service is running correctly:

sudo systemctl status zerotrustkerberos
curl http://localhost:8080/health

Security Note

ZeroTrustKerberosLink implements comprehensive security hardening by default, including input validation, protection against XSS and injection attacks, secure headers, and TLS support for all communications.
